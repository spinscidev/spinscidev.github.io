SpinSci SnapJs

Implementation of Finesse's call control in a miniture html5 UI. SnapJs can be embedded in CRM solutions or served from anywhere. It is extensible by plugins and lightweight in comparison to the default Finesse Desktop.

Building and Using SnapJs

To build from source you need npm 5 or greater.

To build run:

npm install
npm run build

After building you can add the snap.min.js or snap.js to any html file. Also to that html file you must add <div id='snapjs'></div> to where you'd like the snap interface to end up. To get snap to add it's UI and be ready for an agent to sign in you need to call snapJs.init(config) and pass a valid configuration object.

See the SnapJs configuration documentation.

See the SnapJs public interface documentation.

See the Cisco Finesse.js documentation for documentation on how to interact with the user, dialog, reasonCodes, etc...

Plugin Development

A SnapJs plugin must contain an init(config, snap) method for SnapJs's init function to call, start the plugin, and add any extra ui elements. Additionally, a plugin may have an add(snap) method which gets called as soon as the plugin is added and is the best place to modify the SnapJs public interface, or to do anything that needs to happen before init gets called.

It is standard practice for plugins to expect SnapJs be in the global scope of the browser and to add themselves to SnapJs so that they do not clutter the global scope themselves. Plugins should be written as npm packages containing CommonJS modules. Also, they should be written so that they work the same if the built js is added directly to the html in a <script> tag or if they are included in another javascript package using require('<plugin package name>').

Beware of the "new" keywords

For objects provided by the finesse.js such as User or Dialog, never attempt to get a reference to them by calling new User({ id: '<agent id>'}). Finesse.js can only push events to one instance of these objects per browser window context, so if you make a new instance it will disconnect the instance that SnapJs is using. To get a reference to the User object use snapJs.onUserLoad or snapJs.onUserChange.
